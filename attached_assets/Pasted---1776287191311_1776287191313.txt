تمام، خلصت دراسة الكود كامل. خليني أعرض عليك الرؤية الكاملة كخبير.

الوضع الحالي
التطبيق دلوقتي هو CRM أحادي المستأجر (Single-Tenant) - يعني نسخة واحدة بتخدم شركة واحدة. مفيش فصل بيانات بين عملاء مختلفين، ومفيش نظام اشتراكات أو صفحات تسويقية.

الرؤية المقترحة للتحويل لـ SaaS
الفكرة إن المنصة بتقدم:

شات بوت بالذكاء الاصطناعي يدير المحادثات تلقائياً عبر واتساب وماسنجر وانستجرام، يجمعها في مكان واحد، ويحولها لعملاء فعليين من خلال CRM متكامل - لأي بيزنس خدمي بيعمل حملات ومحتاج يرد على الرسائل 24/7

التقسيم المقترح (7 مراحل مترتبة):
المرحلة 1: البنية التحتية للـ Multi-Tenant (الأساس - لازم يكون أول حاجة)
إنشاء جدول companies (الشركات/المستأجرين)
إضافة company_id لكل جداول البيانات (leads, tasks, communications, إلخ)
تعديل كل الـ queries عشان تفلتر بالـ company_id
تحديث نظام الـ Auth عشان يدعم سياق الشركة
ترحيل البيانات الحالية لشركة افتراضية
نظام الـ Roles يتطور: platform_admin (صاحب المنصة) vs company_owner + باقي الأدوار
المرحلة 2: نظام الاشتراكات وتتبع الاستخدام
جدول subscription_plans (الباقات مع feature flags)
جدول subscriptions (اشتراك كل شركة)
حدود الاستخدام (عدد مستخدمين، عدد ليدز، عدد رسائل واتساب)
Middleware يتحقق من حدود الباقة قبل أي عملية
صفحة إدارة الباقات لصاحب المنصة
المرحلة 3: لوحة تحكم صاحب المنصة (Platform Admin Dashboard)
Dashboard بـ KPIs حية: إجمالي الشركات النشطة، MRR، ARR، نسبة التحويل من Trial
صفحة إدارة الشركات (قائمة + تفاصيل + فلاتر + إجراءات)
إجراءات: تعليق شركة، إعادة تفعيل، تغيير باقة، تسجيل دخول كشركة
لوحة الإيرادات: رسوم بيانية MRR، تقسيم حسب الباقة، الفواتير القادمة
نظام التذاكر والدعم الفني
مركز الإشعارات (تسجيل جديد، تجربة بتنتهي، دفعة متأخرة)
المرحلة 4: CRM خاص بصاحب المنصة
صاحب المنصة محتاج نفس الـ CRM عشان يبيع المنصة نفسها
Pipeline خاص بالليدز بتوع المنصة (مش ليدز العملاء)
ده هيستخدم نفس البنية الموجودة بس في سياق الـ Platform Admin
المرحلة 5: الموقع التسويقي العام (Public Website)
صفحة رئيسية احترافية (Hero, Features, Testimonials, FAQ)
صفحة الأسعار مع مقارنة الباقات
صفحات About, Contact
صفحة التسجيل (Multi-step signup)
الصفحات القانونية (Privacy, Terms)
SEO كامل (JSON-LD, Open Graph, Sitemap, robots.txt)
المرحلة 6: Blog CMS
محرر مقالات WYSIWYG مع لوحة تقييم SEO
صفحة المدونة مع فلاتر وبحث
صفحات المقالات الفردية مع مشاركة اجتماعية
إدارة التصنيفات
Sitemap تلقائي
المرحلة 7: معالج الإعداد (Onboarding Wizard)
5 خطوات بعد التسجيل (بيانات الشركة → المستخدمين → الخدمات → ربط واتساب → انتهاء)
جولة تعريفية داخل التطبيق لأول مرة
حفظ التقدم عشان يكمل لاحقاً
نقاط مهمة أضيفها كخبير:
التسعير الذكي: لازم الباقات تكون مبنية على القيمة - عدد المحادثات الآلية شهرياً + عدد الليدز + عدد قنوات التواصل
Trial Period: 14 يوم تجربة مجانية بكل المميزات عشان العميل يجرب ويتعلق
Usage Dashboard لكل عميل: كل شركة تشوف استهلاكها من الحدود (كام رسالة باقية، كام ليد)
الفصل في الـ Routing: /app/* للعملاء، /platform/* لصاحب المنصة، / للموقع التسويقي
White-label مستقبلاً: البنية لازم تسمح إن كل شركة يكون عندها لوجو ولون مخصص
